The Raw Fingerprint Is NEVER Stored Anywhere

 Enrollment (First Time)
1
Employee scans finger on USB device
2
Device generates KEY PAIR
🔑 Private Key → stays on workstation
🌐 Public Key → sent to cloud
3
Cloud stores:
userId + publicKey + deviceId
 Every Clock-In After That
1
Employee places finger on scanner
2
Device signs a challenge using Private Key
3
Signed token sent to cloud
4
Server checks against stored Public Key
5
Match ✅ → Time logged
No match ❌ → Denied
✅ No fingerprint image stored.    ✅ No biometric template in the cloud.

Admin sees where they clocked in

Best for: Small companies

 Option B
Whitelist ⭐ Recommended

Unapproved attempt → denied + admin alert

Best for: Mid-size companies

 Option C
Request-Based (Most Like UKG)

Employee requests access to a location

Admin approves/denies, can be time-limited

Best for: Enterprise

The Detail Most Systems Get Wrong

Scenario: Employee hired Chicago (CST) clocks in at New York (EST)
❌ Wrong: Store local time → hours look wrong on payroll
✅ Right: Always store UTC → display in local timezone
STORE
Always UTC in the database
RECORD
Timezone of the location
DISPLAY
Convert to local time for the UI

{
  "clock_in": "2026-05-08T14:00:00Z",  // UTC in DB
  "timezone": "America/New_York",       // location timezone
  "location": "New York Office"
}
// Chicago HQ admin sees: "9:00 AM CST"
// Employee sees:         "10:00 AM EST"

The Scenarios That Break Other Systems

ScenarioHow We Handle It
Double clock-in same locationBlock — "You are already clocked in here"
Cross-location clock-out✅ Allowed — both locations recorded
Unauthorized locationDeny + real-time alert to admin
Timezone differencesStore UTC, display in local timezone
Scanner unplugged mid-shiftElectron re-detects on reconnect; shift unaffected
Cloud unreachableElectron queues clock events; syncs when back online
Split shift across locationsTwo separate time entries — both valid
Fake / stolen fingerprintChallenge-response crypto — replay attacks blocked

Cloud time tracker with biometric auth, multi-location support, live admin dashboard
